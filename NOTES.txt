🔐 AuthContext — Control Flow
App starts
        ↓
AuthProvider wraps whole app
        ↓
useState checks localStorage
"Is there a saved user?"
        ↓
YES → user = saved user data
      (stays logged in after refresh!)
        ↓
NO → user = null
     (not logged in)
        ↓
Any component calls useAuth()
        ↓
Gets user, login, logout instantly!

━━━━━━━━━━━━━━━━━━━━━━━

LOGIN FUNCTION FLOW:
login(userData) called
        ↓
localStorage.setItem('user', userData)
saves in browser permanently
        ↓
setUser(userData)
updates state
        ↓
whole app re-renders
navbar shows "Hi Esha!"
protected routes unlock!

━━━━━━━━━━━━━━━━━━━━━━━

LOGOUT FUNCTION FLOW:
logout() called
        ↓
localStorage.removeItem('user')
deleted from browser
        ↓
setUser(null)
        ↓
whole app re-renders
navbar shows Login/Register again
protected routes lock!

🛒 CartContext — Control Flow
App starts
        ↓
CartProvider wraps whole app
        ↓
cartItems = [] (empty cart)
        ↓
Any component calls useCart()
Gets cartItems, addToCart, etc!

━━━━━━━━━━━━━━━━━━━━━━━

ADD TO CART FLOW:
User clicks "Add To Cart"
        ↓
addToCart(product) called
        ↓
Is product already in cart?
        ↓
YES → find that item
      increase quantity by 1
      cartItems updated!
        ↓
NO → add new item
     with quantity = 1
     cartItems updated!
        ↓
Navbar cart count updates automatically!
totalItems recalculates!

━━━━━━━━━━━━━━━━━━━━━━━

REMOVE FROM CART FLOW:
User clicks ❌ on cart item
        ↓
removeFromCart(id) called
        ↓
filter removes that item
        ↓
cartItems updated!
totalItems recalculates!

━━━━━━━━━━━━━━━━━━━━━━━

CLEAR CART FLOW:
Order placed successfully
        ↓
clearCart() called
        ↓
cartItems = []
        ↓
Cart is empty again!
Navbar shows 🛒 (0)

🔐 LoginPage — Control Flow
User visits /login
        ↓
LoginPage component loads
        ↓
States initialized:
email = ''
password = ''
error = ''
loading = false
        ↓
User types email
onChange fires
setEmail(value) updates state
        ↓
User types password
onChange fires
setPassword(value) updates state
        ↓
User clicks LOGIN button
        ↓
handleSubmit(e) runs
        ↓
e.preventDefault()
stops page reload
        ↓
setLoading(true)
button shows "Logging in..."
button disabled!
        ↓
axios.post('/api/auth/login')
sends email + password to backend
        ↓
BACKEND PROCESSING:
finds user by email
compares password with bcrypt
        ↓
WRONG CREDENTIALS?
catch block runs
setError("Invalid email or password!")
red error box appears on screen
setLoading(false)
        ↓
CORRECT CREDENTIALS?
data comes back with token
login(data) → saved in context
navigate('/') → goes to home!
setLoading(false)

📝 RegisterPage — Control Flow
User visits /register
        ↓
RegisterPage component loads
        ↓
States initialized:
name = ''
email = ''
password = ''
confirmPassword = ''
error = ''
loading = false
        ↓
User fills all 4 inputs
each onChange updates state
        ↓
User clicks REGISTER button
        ↓
handleSubmit(e) runs
        ↓
e.preventDefault()
stops page reload
        ↓
PASSWORD CHECK:
password === confirmPassword?
        ↓
NO → setError('Passwords do not match!')
     red box shows
     function STOPS (return)
     API never called!
        ↓
YES → continue
        ↓
setLoading(true)
button shows "Creating Account..."
        ↓
axios.post('/api/auth/register')
sends name + email + password
        ↓
BACKEND PROCESSING:
checks if email exists
hashes password with bcrypt
saves new user to MongoDB
sends back user + token
        ↓
EMAIL EXISTS?
catch block runs
setError("Email already registered!")
red box shows
        ↓
SUCCESS?
login(data) → auto logged in!
navigate('/') → goes to home!
